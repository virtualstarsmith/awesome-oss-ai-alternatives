# Contribution Guidelines / 贡献指南

Please note that this project is released with a [Contributor Code of Conduct](code-of-conduct.md). By participating in this project you agree to abide by its terms.

请注意，本项目附带 [贡献者行为准则](code-of-conduct.md)。参与本项目即表示您同意遵守其条款。

---

## Adding a project / 添加项目

Ensure your pull request adheres to the following guidelines:

请确保您的 PR 符合以下准则：

- The project must have a clear **open-source license** (MIT, Apache-2.0, AGPL, GPL, MPL).
  项目必须有明确的**开源协议**。
- The project must be a **clear alternative** to a named commercial/paid AI product.
  必须是某个**明确商业产品**的开源平替。
- The project must be **active in 2025–2026** (recent commits or releases).
  2025–2026 年仍**活跃维护**。
- The project must be **self-hostable, fully local, or BYOK** (bring-your-own-key).
  支持**本地部署 / 自托管 / 自带 Key**。
- Use the following line format:
  `[**org/repo**](https://github.com/org/repo) — EN description. / 中文描述。 *(alt: Commercial Tool)*`
  使用统一格式：`[**org/repo**](https://github.com/org/repo) — 英文亮点. / 中文亮点。 *(alt: 对标产品)*`
- The English description should start with a capital letter and end with a period.
  英文描述以大写字母开头，以句号结尾。
- Add new entries to the **bottom** of the relevant category.
  新条目添加到对应分类的**底部**。
- Search previous suggestions before making a new one, as yours may be a duplicate.
  提交前请先搜索，避免重复。
- Make an individual pull request for each suggestion.
  每个建议**单独提交 PR**。
- Check your spelling and grammar.
  检查拼写和语法。

## Reporting issues / 报告问题

- **Broken link / 链接失效** — Open an issue with the project name and the broken URL.
  请提供项目名和失效链接。
- **Incorrect information / 信息错误** — Open an issue with the correct details.
  请提供正确信息。
- **Project suggestion / 项目推荐** — Open an issue with the GitHub URL, commercial tool it replaces, and open-source license.
  请提供 GitHub 地址、对标的商业产品和开源协议。
